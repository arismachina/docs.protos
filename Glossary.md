# Glossary

← [Back to Home](Home)

| Term | Definition |
|------|-----------|
| **Canvas** | The visual graph workspace in a Protos project, where all nodes (designs, data, models, results) are connected and displayed. |
| **Copilot** | The AI assistant embedded across all Protos features, providing suggestions, surfacing knowledge, and accelerating configuration. |
| **Design freeze** | An immutable, permanently linkable snapshot of a validated design, used for manufacturing handoff. |
| **Design space** | The range of possible parameter combinations for a design, explored via sweeps in Simulation Studio. |
| **Node** | Any artifact on the canvas: a design, data entry, simulation result, model call, parameter, or decision. |
| **PSPP** | Process → Structure → Property → Performance. The reasoning framework underlying Protos, used to trace how manufacturing choices lead to physical outcomes. |
| **Reference field** | A schema field type that links one schema entry to another, creating relational structure across engineering data. |
| **Schema** | A defined structure for a type of engineering data in Protos (e.g. a test result schema, a design parameter schema). |
| **Sweep** | A batch simulation run that varies one or more parameters across a range, returning an output surface. |
| **TRL** | Technology Readiness Level. Protos is optimized for TRL 0–3, the early R&D phase where first-principles reasoning and traceability are most critical. |
| **Trace** | The ability to follow any value in Protos back through its chain of sources to the original reference, experiment, or decision. |
| **Version** | A named snapshot of a design or model at a point in time. Versions are immutable once created. |
